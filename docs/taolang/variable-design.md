# 变量设计哲学

在多数编程语言中，变量是一个从名字到值的简单映射。名字是标签，值是数据，二者之间没有任何可供操作的中间地带。TaoLang 对这一基本假设提出了不同的回答：变量不是被动的容器，而是具有身份、行为和生命周期的活体对象。

这一理念通过"三重嵌套"结构得以实现。

---

## 三层结构

TaoLang 中的每个变量由三个层次构成：

```
+--------------------------------------+
|  第一层：外延层 (Extension Layer)      |  <-- 身份标识
+--------------------------------------+
|  第二层：内涵层 (Intension Layer)      |  <-- 行为与元信息
+--------------------------------------+
|  第三层：数值层 (Value Layer)          |  <-- 实际数据
+--------------------------------------+
```

这三层从外到内，分别回答了三个问题：**这个变量是谁**、**这个变量如何行为**、**这个变量持有什么**。

---

## 第一层：外延层（Extension Layer）

外延层是变量面向程序世界的身份。它包含：

- **变量名** -- 程序中可见的符号名称。
- **作用域信息** -- 变量隶属于哪个空间或函数。
- **可见性** -- `public` / `private` / `protect`。
- **符号表索引** -- 编译器内部的定位标识。

外延层回答的是"这个变量是谁"的问题。在源代码中，当我们写下 `let my_var = 42` 时，`my_var` 这个名字就是外延层的全部。它是变量在人类世界的名片，是符号表中的一个条目，是作用域中的一个位置。

外延层本身不承载任何行为，也不直接接触数据。它是纯粹的标识。

---

## 第二层：内涵层（Intension Layer）

内涵层是 TaoLang 变量设计的核心创新。它位于名字和值之间，构成了一个可编程的中间地带。

内涵层包含：

- **生命周期钩子** -- `on` / `when` 事件处理器，在变量的关键时刻执行自定义逻辑。
- **元数据** -- 类型信息、生命周期标记、属性等描述性信息。
- **内存地址指针** -- 指向数值层中实际数据的指针。
- **委托规则** -- 动态变量的求值逻辑，控制变量在每次访问时如何产生值。
- **生命周期属性** -- 如 `alive[auto]`、`alive[forever]` 等，决定变量存活策略。

传统语言中，从名字到值是一次直达的跳转。TaoLang 在这两者之间插入了内涵层，使得每一次对变量的访问、修改、创建和销毁都经过一个可观察、可拦截的中间环节。

内涵层赋予变量以下能力：

- **自我感知** -- 变量可以"知道"自己何时被读取、何时被修改。
- **自我保护** -- 通过 `when` 钩子拦截非法修改，拒绝不合理的赋值。
- **自我管理** -- 控制自身的生命周期，在恰当的时机执行资源清理。
- **响应式求值** -- 通过委托机制，在每次访问时动态计算值，而非返回静态存储。

正是内涵层的存在，使得变量从被动的数据容器升格为主动的参与者。

---

## 第三层：数值层（Value Layer）

数值层是变量最终持有的实际数据：

- **标量值** -- `int`、`float`、`bool`、`str`、`tuple` 等基本类型。
- **引用值** -- 对象、数组、结构体等复合类型。
- **内存中的实际位序列** -- 运行时真正存储在内存中的比特。

数值层是变量存在的根本理由。无论外延层如何命名、内涵层如何加工，最终程序需要的是这里存放的数据。数值层的纯粹性在于它只关心"是什么"，不关心"叫什么"或"怎么行为"。

---

## 与传统变量的对比

### 传统变量：两层结构

```
名字 --> 值
```

在传统语言中，变量是名字到值的直接映射。生命周期管理依赖语言运行时（垃圾回收器、引用计数），事件监听需要额外的设计模式（观察者模式、发布-订阅），行为约束散落在使用变量的各个角落。

变量本身是沉默的。它不知道自己何时被读取，不关心赋给自己的值是否合理，也无法在消亡前做出任何反应。

### TaoLang 变量：三层结构

```
名字 --> 内涵（钩子 + 元数据 + 指针） --> 值
```

中间的内涵层打破了这种沉默。变量不再是两个端点之间的一根直线，而是一条经过中间站的路径。每次对变量的操作都经过内涵层的"关卡"，在那里可以被观察、被拦截、被增强。

这一差异的本质是：传统变量是数据的容器，TaoLang 变量是数据的守护者。

---

## 三层结构的核心优势

### 细粒度生命周期控制

变量的每个生命阶段——创建、访问、修改、删除、释放——都是可挂载逻辑的事件节点。开发者不再需要在变量的每个使用点手动添加检查和日志，而是在声明时一次性定义变量在各阶段的行为。

### 原生响应式编程

通过委托机制，变量可以在每次被访问时动态求值，而非返回一个静态存储的值。这使得响应式编程成为语言内建的能力，无需引入外部框架或库。

### 行为与数据统一

在传统编程中，验证逻辑在一处，日志记录在另一处，资源清理在第三处。TaoLang 通过内涵层将这些行为直接附着在变量自身之上，使得数据和围绕数据的行为在定义点就紧密结合。

### 内存管理透明化

内涵层持有指向数值层的内存地址指针，使得内存操作可以在钩子中显式进行。开发者在需要时可以直接控制内存行为，而非完全依赖语言运行时的隐式决策。

### 零成本抽象

三层结构的开销是按需付出的。如果一个变量没有定义任何钩子，编译器会将内涵层完全优化掉，使其在编译后的表现与传统两层变量完全一致。只有当开发者显式使用了钩子和内涵层功能时，才会生成相应的管理代码。

简单变量退化为传统结构，复杂变量按需承担开销。

## 元数据系统

内涵层维护丰富的元数据，可通过 `of` 关键字查询：

```tao
let x = 42
println(x of type)        // int
println(x of symbol)      // x
println(x of space)       // 当前空间
println(x of alive[auto]) // true / false
```

元数据使得变量的自省成为一等公民。程序可以在运行时查询变量的类型、名称、所属空间和生命周期属性，而这些信息来自内涵层对变量身份和状态的完整记录。

## 解引用与层级访问

TaoLang 提供语法手段直接访问变量的不同层级：

| 表达式 | 含义 |
|--------|------|
| `a` | 通过外延层访问，经由内涵层到达数值 |
| `*a` | 数值层指针，直接操作底层数据 |
| `[a]` | 变量的完整实体（symbol + hooks + value） |
| `*(*a)` | 二级解引用，穿透引用链 |

这些语法手段在日常编程中很少需要，但在编写底层逻辑、自定义内存管理和高级钩子时不可或缺。

### `[x]` -- 变量的完整实体

`[x]` 语法访问的是变量 `x` 的**完整实体**（"所有"），它是一个包含三个组成部分的整体：

| 组成部分 | 对应层级 | 说明 |
|----------|---------|------|
| **symbol**（符号） | 外延层 | 变量名绑定，是访问实体的入口 |
| **hooks**（钩子） | 内涵层 | 生命周期钩子与元数据 |
| **value**（值） | 数值层 | 变量实际存储的数据 |

通过 `of` 关键字可以查询完整实体的各组成部分：

```tao
let x = 42
[x] of symbol  // x（符号名称）
[x] of type    // int（类型信息，来自内涵层）
[x] of space   // 当前所属空间
```

#### `free(x)` 与 `free([x])` 的区别

`free(x)` 和 `free([x])` 操作的目标不同，这是理解 TaoLang 内存模型的关键：

- **`free(x)`** -- 释放符号：仅解除 `x` 这个 symbol 与底层实体的绑定。如果同一实体还有其他 symbol 指向它（如 `y`），则实体继续存活，可通过 `y` 访问；如果已无任何 symbol 指向该实体，编译器自动优化释放整个实体。
- **`free([x])`** -- 释放完整实体：强制释放 `x` 的完整实体（symbol + hooks + value 所有层），无论是否还有其他 symbol 指向它。

```tao
let x = 42
let y = x       // y 与 x 指向同一实体

free(x)          // 仅释放 x 的 symbol，y 仍可访问该实体
println(y)       // 42（实体仍存活）

free([y])        // 释放完整实体，所有层被回收
```

这一区分使得 TaoLang 在多符号共享实体的场景下具备精确的内存控制能力。

## 设计哲学总结

TaoLang 的三重嵌套变量设计体现了以下核心理念：

**变量即生命** -- 变量不是静态的存储槽位，而是拥有完整生命周期的实体。从创建到消亡，每个阶段都是可观察、可响应的。

**行为与数据统一** -- 通过内涵层的钩子系统，行为直接附着在数据之上，而非散落在程序各处。数据和围绕数据的逻辑在定义点就成为一个整体。

**显式生命周期** -- 生命周期不再是语言运行时的黑箱。开发者可以观察、控制、介入变量生命周期的每个阶段。

**零成本抽象** -- 复杂性按需付出。不使用内涵层功能的变量没有任何额外开销。

**声明式响应** -- 通过钩子声明"当某事发生时应如何响应"，而非在每个操作点命令式地编写回调。

这五项理念的交汇，使得 TaoLang 在语言层面原生支持 RAII、观察者模式、响应式编程、面向切面编程和约束编程。这不是对现有模式的简单叠加，而是通过三层结构将它们统一在一个一致的模型之下——这正是 RSOP 范式在变量层面的具体体现。
